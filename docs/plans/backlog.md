# Backlog

Compact seeds. Keep enough context that a future reader can recover the trigger, constraint, and likely shape without archaeology. Promote a seed to its own plan once it needs sections, ordered units, acceptance criteria, or design rationale that no longer fits comfortably here; delete the seed here when it graduates.

- **Surface contributions + workspace layout** — done: features export `defineSurface` contributions from their workspace dirs; `layout.tsx` composes the flat list and `SurfaceMount` hands each surface its typed channel client.
- **Typed channel events** — done: shared `ChannelContract`s drive backend handlers (`withHandlers`), typed frontend clients (`createChannelClient`), and typed event publishers (`FeatureEventPublisherFactory`); events are schema-validated on both publish and subscribe.
- **Discovery-fed surface composition** — the layout is still a build-time import list; discovered features contributing surfaces need a runtime surface registry in the workspace, and foreign/generated surface code additionally needs the iframe surface transport (the containment mode shelved when the Host/Workspace iframe collapsed — see [workspace-runtime-foundation](./archive/workspace-runtime-foundation.md)). Includes deriving canvas open keys from visible surfaces (still `openCanvasKeys = ["main"]`) and, later, feature-agent link metadata. Gated on [feature-loading-convergence](./archive/feature-loading-convergence.md) for the backend half.
- **`uix-core` embedded-pi config** — orientation block + doc map + smoke-test tools injected into the cockpit's agent.
- **Agent tool contribution from extensions** — done: extensions register pi tools into the owned session via `AgentToolRegistry`.
- **File watcher service** — cockpit-owned watcher; extensions register glob → callback.
- **Default conversation feature** — chat is a renderer-only feature (`features/chat/workspace/`) contributing `chatSurface` with no backend contributions (agent channels are substrate-owned). Removability now gates on discovery-fed surface composition — the surface exists as a contribution but the layout list is a build-time import.
- **Canvas as a default feature** — canvas has full backend contributions (`FeatureDefinition`, channels, tools, turn state, agent context) and contributes `canvasSurface`. Same removability gate as chat: discovery-fed surface composition.
- **App dirs + start picker** — a UIX App is a workspace composition rooted in a user-chosen directory, anywhere on disk: feature packages live visibly in the dir, state (pi session, canvas store) under its `.uix/`, and `resolveWorkspace()`'s `process.cwd()` placeholder is replaced by the picked dir (the seam `workspace.ts` already reserves for "a future project-picker"). On start, a small shell modal creates a new App (choose dir + scaffold defaults) or reopens a recent one (recents persisted in Electron `userData`); one BrowserWindow per open App matches the existing multi-workspace = multi-window model. Discovery roots become app-relative; whether a cross-app `~/.uix/features/` root survives is an open call.
- **Default features scaffold into new Apps** — chat and canvas are purely user space: shipped inside the binary as source-form template packages (e.g. `resourcesPath/default-features/`), copied into an App dir at creation time, then loaded through ordinary discovery — deletable per App, agent-editable with `/reload` applying changes, and the living worked example for agents authoring features. Today they compile into the app bundle via `bundled.ts`/`layout.tsx` static imports, which satisfies "default" but not "user space" or "readable example." Stamping at App creation (not a central materialized copy) kills the template-update-policy problem: new Apps get new templates, existing Apps keep theirs. Gated on discovery-fed surface composition (their frontends must load at runtime) plus `@uix/api` becoming value-importable at feature runtime (real package per the api-type-alias upgrade path, or a loader alias table exposing a blessed shared dep set: `@uix/api`, `react`, `typebox`). Once scaffolding lands, `bundled.ts` must die rather than become a fallback — a bundled fallback would resurrect a default the user deliberately deleted from their App.
- **Session inspector / debug mode** — menu toggle rendering `display: false` custom messages (and other hidden entry streams) in the conversation pane; renderer-only, the transcript items already carry customType/content/details.
- **User-action event log on CustomEntry** — append-only `pi.appendEntry` streams folded by D3 reducers; model-invisible, branch-aware, no second database. Keep low-frequency (whole session file loads into memory); leaf discipline unresolved for concurrent writers.
- **Fan-out one-off mode** — prepared root per app variant (system prompt + tool contracts + vocabulary + seeded state entries), fork per invocation, never rejoins; sibling branches make every run inspectable and identical prefixes align with provider prompt caching. Needs lifecycle design (fork-per-invocation creation/disposal, concurrency), not new messaging.
- **Docs + examples** — flesh out `src/docs/` stubs and seed `examples/` as primitives land.
