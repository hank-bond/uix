# Current state

The current state of the system: what's built, how the code is shaped, and the open questions we haven't resolved. Living architecture docs always describe HEAD and are rewritten freely to match the code. This is the dev-facing architecture-of-record; the vision lives in the repo-root [`../../AGENTS.md`](../../AGENTS.md), and the user-facing substrate reference that ships lives in [`../../src/docs/`](../../src/docs/).

**Built:**

- Electron + electron-vite scaffold (`electron.vite.config.ts`, `src/main`, `src/preload`, `src/renderer`).
- Typed IPC scaffold (`src/shared/ipc.ts`, preload bridge).
- Pi `createAgentSession` driver in the main process (`src/main/agent/driver.ts`).
- Lifetime-scoped disposables and lifecycle helpers (`src/main/lifecycle.ts`). See [conventions](./conventions.md).
- Chat feature surface (`src/features/chat/workspace/`) with scoped chat CSS, default chat blocks, exact first-party canvas tool block renderers, and a local Iosevka code font asset. Renderer-only: agent channels are substrate-owned.
- **Single-page workspace renderer** (`src/renderer/workspace/`) — one page is one workspace; multi-workspace is multi-BrowserWindow; no Host frame, iframe, or postMessage bridge (collapsed in `8adeb7d` — see [workspace-runtime-foundation](../plans/archive/workspace-runtime-foundation.md)). Features export surface definitions via `defineSurface` from their `workspace/` dirs; `layout.tsx` composes the flat `SurfaceContribution` list and `SurfaceMount` mints each surface's typed channel client. `WorkspaceClient` wraps the preload `ChannelTransport` (`window.channels`); workspace code stays web-compatible behind it.
- **Typed channel substrate** — features declare schema-only `ChannelContract`s in `shared/`; the backend merges executable handlers via `withHandlers` (requests/responses parsed both directions by `ChannelRegistry`, `src/main/channels/registry.ts`); the frontend derives fully typed clients via `createChannelClient` (`@uix/api/workspace`, events validated on subscribe); backend feature code publishes through contract-derived typed publishers minted per feature by `FeatureEventPublisherFactory` — presenting a contract is the only way to obtain publish capability.
- **Feature contribution system** (`src/main/features/`) — `FeatureDefinition` (`id`, optional `context(ctx)` hook, `contribute(ctx)`) feeding facet registries for resources, channels, agent tools, turn state, and agent context; `registerFeatureContributions` bags each facet registration per feature. Canvas is the first full `FeatureDefinition`; `bundled.ts` is the hardcoded inventory registered from `main/index.ts` into `appBag`. Substrate agent channels (prompt/history/events) register through the same `ChannelRegistry` under the `agent` id.
- **Feature loader** (`src/main/features/manifest.ts`, `loader.ts`) — one load pass activates the whole composition: bundled defaults first (claiming their ids), then the workspace manifest's entries in manifest order. `uix.workspace.json` (`name` + ordered feature entry-file refs, relative to the manifest or absolute) replaces auto-discovery entirely; each jiti-loaded entry is validated as a default-exported `FeatureDefinition` (shape, id grammar, reserved `agent`/`uix` ids, duplicate ids) and registered through `registerFeatureContributions` under a per-feature bag in `featuresBag`. `/reload` re-reads the manifest and re-runs everything — bundled included, with fresh context/bags — then delegates to pi; a throwing feature lands in `failed[]` without aborting the pass, while a malformed manifest rejects the pass before anything is cleared (startup falls back to bundled-only so the pilot can fix it and `/reload`). Transitional until M2/M3 of the plan: the manifest is looked up at the cwd and is optional. ([workspace-manifest-not-discovery](../decisions/2026-07-02-workspace-manifest-not-discovery.md), [features-are-the-loadable-unit](../decisions/2026-07-01-features-are-the-loadable-unit.md), [workspace-manifest-and-picker](../plans/workspace-manifest-and-picker.md).)
- **Resource routes substrate** — a single `uix-resource://` custom protocol with route-based dispatch replaces per-resource scheme registration. `src/shared/resource-routes.ts` defines the route model (`path`, `query`, `origin`), normalization, branded `ResourceUrl` encoding, and URL decoding. `src/main/resources/registry.ts` stores `NormalizedResourceRoute` registrations in a Map and dispatches incoming requests by decoding URLs against each route pattern, handing pre-parsed `{ params, query }` to feature handlers via `ResourceRequestContext`. Feature authors declare routes with `createResourceAddressBuilder` from `@uix/api/resources`, which returns `{ route, url(), origin() }` — `route` goes into the contribution, `url()`/`origin()` are consumed by renderer code. The builder is a transitional hybrid; it dissolves into substrate-owned transport when surface contributions land. See [resource-routes](../plans/archive/resource-routes.md).
- Canvas surface: feature-origin `uix-resource://canvas.<workspace>/doc/...` resource route (declared by `createResourceAddressBuilder`), key-addressed mutable latest store, immutable JSON canvas snapshots with anchor-state meta, private turn-state contributions for per-submit and post-agent `uix.turn-state` snapshot refs, snapshot-derived `<canvas.canvas-diff>` agent context, anchored canvas tools, whole-document refresh, and an injected writeback shim for form/control and explicit `contenteditable` edits (`src/features/canvas/backend/`, `src/features/canvas/workspace/`, `src/features/canvas/shared/`, `src/main/turn-state/`, `src/main/agent-context/`). Writeback postMessage listener still hand-rolls origin/source/key checks (it forwards into the typed channel client) — substrate-owned message routing for nested iframes remains deferred until the iframe surface transport exists. See [canvas-stage-one](../decisions/2026-05-31-canvas-stage-one.md), [canvas-data-channel](../design/canvas-data-channel.md), and [pane-and-file-versioning](../design/pane-and-file-versioning.md).

**In flight:** turn state (`src/main/turn-state/`) owns appending opaque per-feature refs at run boundaries and agent context (`src/main/agent-context/`) materializes model-visible `<uix-state>` sections from them; the turn-state contract still needs the navigation half — preview/restore callbacks beside prepare — which unblocks branch preview/restore and folds in anchor rehydration ([persistence-and-session-foundation](../plans/persistence-and-session-foundation.md)). Durable transcript identity ([durable-transcript-identity](../plans/durable-transcript-identity.md)) is the next substrate needed before interactive or stateful chat blocks. Canvas edits its working copy through `CanvasDocumentBuffer` (a purpose-owned working copy over the store, not a transport; "channel" names only message conduits). Filesystem-tool parity and surface versioning/rollback are tracked in [canvas-data-channel](../design/canvas-data-channel.md) and [pane-and-file-versioning](../design/pane-and-file-versioning.md).

**Deferred from resource routes:** two items captured in [open questions](./open-questions.md) — substrate-owned postMessage origin/source filtering, and the `createResourceAddressBuilder` dissolving when surface contributions land (the builder's `url()` and `origin()` become substrate-internal at that point).

What's _next_ lives in the plans backlog ([`../plans/backlog.md`](../plans/backlog.md)); per-decision history is in [`../decisions/`](../decisions/).
